## 二、布局排版分析与已实施优化

## 三、功能完整性评估

### 3.1 已完成闭环的功能

```
采集 → 解析 → 确认 → 写入画像/日程/账单 → 提醒 → 履约 → 对账
```

- **采集收件箱**：支持文字/语音/截图/聊天/账单 5 种来源
- **AI 解析**：本地规则 + OpenAI 云端双通道
- **确认中心**：可编辑、确认、驳回
- **提醒升级**：Level 1 → 15分钟 → 短信 → 语音
- **履约方案**：生日/节日/差旅三类方案
- **隐私合规**：PII 脱敏 + 6 项开关 + 数据导出/删除
- **运营兜底**：SLA/重试/告警/补录/批量处理

### 3.2 功能缺口分析

| 缺口 | 商业影响 | 优先级 |
|------|----------|--------|
| 无真实支付/下单 | 无法产生直接营收 | P1（V2） |
| 无移动端 App | 无法读取短信/录音 | P1（原生壳） |
| 无多用户/团队 | 仅限个人使用 | P2 |
| 无数据图表 | 复盘体验弱 | P2 |
| 无日历月视图 | 日程管理不便 | P2 |
| 无真实 OCR/ASR | 采集能力受限 | P1 |
| 无真实通知通道 | 提醒无法触达 | P1 |
| 无订阅付费 | 无商业化闭环 | P2 |
| 无微信/支付宝集成 | 无法触达国内用户 | P2 |
| 无 AI 对话交互 | 缺少核心 AI 体验 | P1 |

## 四、商业迭代方向规划

### Phase 1：产品化打磨（4-6周）

**目标**：从 MVP 升级为可内测的 Alpha 版本

| 迭代项 | 描述 | 预期产出 |
|--------|------|----------|
| **AI 对话助手** | 接入 LLM 流式对话，支持自然语言管理日程/人脉/账单 | 核心差异化体验 |
| **日历月视图** | 实现月/周/日切换，支持拖拽调整 | 日程管理可用性 |
| **数据可视化** | 饼图/折线图展示支出流向、预算使用率 | 复盘体验升级 |
| **暗色模式** | 增加主题切换，完善暗色适配 | 视觉品质提升 |
| **新手引导** | 3步引导：采集→确认→方案 | 降低上手门槛 |
| **组件拆分** | 将 3120 行单文件拆为 7 个模块组件 | 工程可维护性 |
| **键盘快捷键** | `Cmd+K` 全局搜索、`N` 新建采集 | 效率提升 |
| **PDF 导出** | 账单复盘/履约方案导出 PDF | 商务场景刚需 |

### Phase 2：真实服务接入（6-8周）

**目标**：接入真实服务商，形成可运营闭环

| 迭代项 | 描述 | 预期产出 |
|--------|------|----------|
| **OCR 接入** | 接入阿里云/腾讯云 OCR，截图/账单自动识别 | 采集能力落地 |
| **ASR 接入** | 语音转文字，长按录音自动采集 | 语音采集落地 |
| **通知通道** | 阿里云 SMS/Voice 正式接入 | 提醒触达用户 |
| **Web Push** | VAPID 配置 + Service Worker 推送 | 浏览器推送 |
| **Supabase 全量** | Auth + Storage + RLS + Cron 全量上线 | 云端数据闭环 |
| **联盟 API** | 京东/淘宝/美团真实订单拉单 | 履约对账落地 |
| **合规引擎** | 动态合规规则 + 标签合成限额 | 商务合规保障 |

### Phase 3：商业化闭环（8-12周）

**目标**：建立订阅付费 + CPS 佣金营收模型

| 迭代项 | 描述 | 预期产出 |
|--------|------|----------|
| **订阅体系** | 体验版/专业版/高管版三档 | 直接营收 |
| **支付接入** | 微信支付/支付宝订阅 | 付费闭环 |
| **CPS 佣金** | 履约跳转佣金归因 + 结算 | 间接营收 |
| **发票系统** | 订阅发票 + CPS 发票 | 企业用户刚需 |
| **权益计量** | 实时扣减 + 超额提醒 + 升级引导 | 付费转化 |
| **家庭共享** | 家庭版多人协作 + 权限管理 | 用户增长 |

### Phase 4：移动端 + 智能化（12-16周）

**目标**：移动端上线 + AI 能力深度整合

| 迭代项 | 描述 | 预期产出 |
|--------|------|----------|
| **React Native 壳** | 短信读取 + 长按录音 + 附件上传 | 原生采集能力 |
| **AI 记忆引擎** | embedding + pgvector 语义召回 | 智能推荐基础 |
| **智能推荐** | 基于记忆+日历+偏好的礼物/餐厅推荐 | 核心 AI 价值 |
| **语音助手** | 语音对话管理日程/查询/提醒 | 交互升级 |
| **PWA 增强** | 离线可用 + 后台同步 | 弱网体验 |
| **多语言** | 英文/日文国际化 | 海外市场 |

## 六、技术债务与工程优化

| 债务项 | 当前状态 | 建议方案 | 优先级 |
|--------|----------|----------|--------|
| 单文件 3120 行 | `liji-app.tsx` 过大 | 拆分为 7 个 Section 组件 + 共享 hooks | P1 |
| 无状态管理 | useState 散落 | 引入 Zustand/Jotai 管理全局状态 | P2 |
| 无 E2E 覆盖 | Playwright 配置就绪但无用例 | 补充 7 个模块核心流程 E2E | P2 |
| 无 CI/CD | 无 GitHub Actions | 补充 lint + typecheck + test + build | P1 |
| 无错误边界 | React Error Boundary 缺失 | 增加全局错误边界 + 降级 UI | P1 |
| 无性能监控 | 无 Web Vitals 采集 | 接入 Vercel Analytics 或自定义 | P2 |
| 无国际化 | 硬编码中文 | 引入 next-intl 或 i18next | P3 |
